Desenho da Solução:

Arquitetura

Fonte dos Dados (API ou banco de dados):

As informações de transações serão enviadas para uma API ou um banco de dados criado pela equipe de engenharia de software. Esses dados serão processados em tempo real pela solução proposta.

Ingestão de Dados em Tempo Real:

Utilizaremos ferramentas como o Azure E H ou Kafka para captar e gerenciar as transações à medida que ocorrem, permitindo o processamento imediato.

Processamento dos Dados:

O Databricks, em conjunto com a tecnologia Spark SS, será responsável pelo tratamento dos dados em tempo real, realizando as transformações necessárias para que os dados estejam prontos para serem utilizados.

Armazenamento de Dados:

Os dados serão armazenados no Delta Lake, do Azure. Esse armazenamento será organizado nas camadas, Bronze, Silver e Gold, assegurando a integridade dos dados.

Consumo e Visualização:

Power BI ou o Databricks SQL serão usadas para gerar consultas e painéis de visualização, mostrando informações em tempo real.
